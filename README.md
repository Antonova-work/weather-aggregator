# 🌤 Weather Aggregator

**Weather Aggregator** — это высокопроизводительный микросервис на Python, который собирает данные о погоде из нескольких независимых источников в режиме реального времени и предоставляет усредненный результат. Это позволяет получить более точные и надежные данные, нивелируя возможные погрешности отдельных погодных сервисов.

## 🚀 Стек технологий

Проект построен с использованием современного стека для обеспечения максимальной скорости разработки и работы приложения:

- **Язык:** Python 3.12+
- **Фреймворк:** [FastAPI](https://fastapi.tiangolo.com/) — для создания быстрого и типизированного REST API.
- **Управление зависимостями:** [uv](https://github.com/astral-sh/uv) — сверхбыстрый менеджер пакетов и виртуальных окружений.
- **Асинхронность:** `asyncio` и [HTTPX](https://www.python-httpx.org/) — для параллельного выполнения запросов к внешним API.
- **Валидация данных:** [Pydantic](https://docs.pydantic.dev/) — для строгой типизации моделей данных.
- **Контейнеризация:** Docker — для обеспечения идентичности среды разработки и продакшена.

## 🏗 Архитектура

Проект следует принципам многослойной архитектуры, что делает код легко расширяемым и поддерживаемым:

- `app/api/` — **Слой API**: Определение эндпоинтов, обработка HTTP-запросов и возвращение ответов.
- `app/domain/` — **Слой Домена**: Описание базовых моделей данных (`WeatherInfo`, `AggregatedWeather`), которые используются во всем приложении.
- `app/services/` — **Слой Бизнес-логики**: Сервис-агрегатор, который координирует запросы к клиентам и вычисляет средние значения показателей.
- `app/infrastructure/` — **Слой Инфраструктуры**: Реализация клиентов для взаимодействия с внешними API (Open-Meteo, wttr.in, 7Timer!, MET Norway).
- `app/frontend/` — Простой пользовательский интерфейс для визуализации работы сервиса.

## 🛠 Функционал

Сервис предоставляет следующие возможности:
- **Агрегация данных**: Одновременный опрос четырех погодных сервисов.
- **Вычисление средних значений**: Автоматический расчет среднего арифметического для:
    - Температуры
    - Влажности
    - Скорости ветра
    - Атмосферного давления
    - Облачности
- **Отказоустойчивость**: Если один или несколько сервисов недоступны, приложение продолжает работу с данными от оставшихся источников.
- **Web-интерфейс**: Встроенная HTML-страница для удобного ввода координат и получения результата.

## 📦 Инструкция по запуску

### Вариант А: Запуск из исходного кода

Для запуска потребуется установленный менеджер пакетов `uv`.

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Antonova-work/weather-aggregator.git
   cd weather-aggregator
   ```

2. **Установите зависимости:**
   ```bash
   uv sync
   ```

3. **Запустите приложение:**
   ```bash
   uv run uvicorn app.main:app --reload --port 8000
   ```
   После этого приложение будет доступно по адресу: `http://localhost:8000`

### Вариант Б: Запуск через Docker

Если у вас установлен Docker, вы можете запустить сервис одной командой, используя готовый образ из Docker Hub.

1. **Загрузите и запустите контейнер:**
   ```bash
   docker run -p 8000:8000 kuzinatra/weather-aggregator
   ```

2. **Откройте браузер:**
   Перейдите по адресу `http://localhost:8000`

## 📖 API Reference

### Получение агрегированной погоды
`GET /api/weather`

**Параметры запроса:**
- `lat` (float): Широта (latitude)
- `lon` (float): Долгота (longitude)

**Пример ответа:**
```json
{
  "lat": 55.7558,
  "lon": 37.6173,
  "average_temperature": 12.5,
  "average_humidity": 65.0,
  "average_wind_speed": 4.2,
  "average_pressure": 1013.0,
  "average_cloud_cover": 40.0,
  "raw_data": [
    {

      "humidity": 60,
      "wind_speed": 4.0,
      "pressure": 1012,
      "cloud_cover": 30,
      "provider_name": "Open-Meteo"
    },
    ...
  ]
